# API 逆向探索检查清单

## 阶段1: 侦察

- [ ] 页面截图，确认数据展示方式
- [ ] 确认是 DOM 渲染还是 Canvas 渲染
- [ ] 识别页面上的筛选条件（类目/日期/维度等）
- [ ] Performance API 获取 API 请求列表
- [ ] 筛选出可能的数据 API（排除埋点/静态资源）
- [ ] 下载 JS bundle 搜索 API 常量映射
- [ ] 确认目标 API endpoint

## 阶段2: 逆向

- [ ] 调用配置/常量接口获取字段定义
- [ ] JS 源码搜索 API 调用处的参数构造
- [ ] 确认每个参数：名称、类型、必填/可选、来源
- [ ] ⚠️ 测试参数名大小写变体
- [ ] 发送最小参数集试探请求
- [ ] 逐步加参数验证返回变化
- [ ] 确认是否需要多次请求覆盖全部数据

## 阶段3: 验证

- [ ] API 返回数据与页面显示对比一致
- [ ] 不同参数组合测试正常
- [ ] 记录完整响应 JSON 结构
- [ ] 记录所有字段含义

## 阶段4: 生成

- [ ] 生成 workflow.json
- [ ] 生成 SKILL.md（含所有"坑"的标注）
